# 世界观生成功能 — 优化方案与流程设计

## 一、当前功能现状分析

### 已有能力

| 层面 | 现状 |
|------|------|
| **前端** | `WorldGenerator.tsx` — 单页表单，填写名称/描述/类型/复杂度后一键生成 |
| **前端** | `WorldList.tsx` — 世界观列表展示（名称 + 描述 + 简要信息） |
| **后端** | `WorldService.createWorldGenerateStream` — LLM 单次 prompt 生成 JSON，写入数据库 |
| **后端** | `WorldService.createRefineStream` — 对单个维度做 light / deep 精炼（有 API，前端未接入） |
| **数据库** | `World` 模型包含 10 个字符串维度字段（geography, cultures, magicSystem, politics, races, religions, technology, conflicts, background, description） |
| **素材库** | `WorldPropertyLibrary` 表已建好，但未投入使用 |
| **图** | `worldBuildingGraph.ts` 为 stub，未实际编排多步骤 |

### 核心不足

1. **一次性生成，缺少引导** — 用户填写简短描述后 AI 一口气输出全部维度，没有循序渐进的交互
2. **维度选择在前端写死** — 5 个 dimension 全部硬编码 `true`，用户无法按需勾选
3. **无一致性校验** — 各维度间可能自相矛盾（如"无魔法世界"却生成了魔法体系）
4. **世界与小说脱节** — `World` 与 `Novel` 表无外键关联，世界设定无法自动注入写作上下文
5. **精炼流程无 UI** — 后端已支持 refine，前端没有入口
6. **素材库未激活** — `WorldPropertyLibrary` 表闲置
7. **无版本/历史管理** — 每次修改直接覆盖，无法回溯

## 二、理想世界观生成流程设计

### 总体流程图

```
[1. 灵感捕获] → [2. 类型选择 & 模板] → [3. 核心规则定义]
       ↓                                         ↓
[4. 分层维度逐步构建] ←→ [5. AI 交互式问答深化]
       ↓
[6. 一致性校验 & 冲突检测] → [7. 世界观总览 & 可视化]
       ↓
[8. 精炼 & 迭代] ←→ [9. 世界观关联小说/角色]
       ↓
[10. 导出 & 版本存档]
```

---

### 阶段 1：灵感捕获（Inspiration Capture）

**目的**：用最低门槛帮助用户表达模糊想法。

- 提供"自由输入"模式：用户用一段自然语言描述想象中的世界（哪怕只有一句话）
- AI 根据输入提取关键词，生成 **世界概念卡片**（包含核心意象、情绪基调、关键元素）
- 提供"随机灵感"按钮：从经典世界观设定中随机抽取元素组合，激发创意
- 支持参考作品输入：用户可输入"类似《斗破苍穹》的世界"，AI 分析参考作品的世界特征作为起点

**产出**：一份初步的世界概念摘要（3-5 句话），作为后续生成的种子。

---

### 阶段 2：类型选择 & 模板系统（Genre Template）

**目的**：根据小说类型提供结构化的世界观框架。

- 预置常见类型模板：
  - **东方玄幻** → 侧重修炼体系、宗门势力、天地法则
  - **仙侠** → 侧重仙凡分界、飞升体系、天道规则
  - **都市异能** → 侧重隐秘组织、能力等级、社会融合
  - **科幻** → 侧重技术树、星际政治、AI/人类关系
  - **西方奇幻** → 侧重种族、魔法学院、神话体系
  - **末日废土** → 侧重生存规则、变异体系、资源分布
  - **历史架空** → 侧重历史节点偏转、势力重组
  - **赛博朋克** → 侧重公司统治、义体改造、数字空间
  - **自定义** → 用户自由组合维度
- 每个模板预定义：
  - 推荐维度及其权重（哪些必填，哪些可选）
  - 该类型的经典元素清单（可作为 checkbox 供用户勾选）
  - 该类型常见的世界观"坑"提醒（如玄幻境界划分不能太多、科幻设定需注意技术自洽等）

**产出**：确定世界类型 + 激活的维度列表 + 用户选择的特色元素。

---

### 阶段 3：核心规则定义（Core Rules / Axioms）

**目的**：先确立世界的"宪法"——不可违背的基础规则，再往下填充细节。

建议引导用户回答以下核心问题（AI 辅助生成建议答案，用户确认或修改）：

1. **力量来源与上限** — 这个世界的力量从何而来？最强者能做到什么？
2. **核心限制/代价** — 力量的使用有什么代价？什么是绝对做不到的？
3. **社会运转逻辑** — 谁掌权？权力如何流动？普通人如何生存？
4. **世界观基调** — 整体偏光明/黑暗/灰色？主角需要对抗什么？
5. **独特卖点** — 这个世界最与众不同的一点是什么？

**产出**：3-5 条核心公理（Axioms），后续所有维度生成以此为"宪法"约束。

---

### 阶段 4：分层维度逐步构建（Layered Dimension Building）

**目的**：不再一次性输出所有维度，而是逐层构建、层层递进。

建议分层顺序（每层生成后用户可审阅、修改、再进入下一层）：

| 层级 | 维度 | 说明 |
|------|------|------|
| **L1 基础层** | 世界观背景、地理环境 | 时空框架、大陆/星球/城市分布、气候、关键地标 |
| **L2 力量层** | 修炼/魔法/科技体系 | 力量分级、晋升路径、核心资源、技术水平 |
| **L3 社会层** | 种族、势力、政治 | 主要种族/文明、国家/组织/宗门、权力结构 |
| **L4 文化层** | 宗教、习俗、经济 | 信仰体系、文化传统、货币/交易体系 |
| **L5 历史层** | 历史事件、时间线 | 创世传说、关键转折点、当前纪元定位 |
| **L6 冲突层** | 核心矛盾、张力 | 种族冲突、阶级矛盾、资源争夺、暗线伏笔 |

**关键设计**：
- 每层生成时，AI 自动参考已确定的上层设定，保证上下文连贯
- 用户可随时跳回任何层修改，系统自动标记受影响的下游维度并提示用户是否需要重新生成
- 每层生成后附带"一致性摘要"——指出本层与已有设定的关联点

---

### 阶段 5：AI 交互式问答深化（Interactive Q&A Deepening）

**目的**：通过 AI 提问，帮助用户发现设定盲区、丰富细节。

- AI 分析已有设定，自动生成"深化问题清单"，例如：
  - "你提到斗气大陆有五大帝国，它们之间目前是和平还是战争状态？"
  - "修炼者在社会中的占比是多少？普通人如何看待修炼者？"
  - "斗气九级之后是否还有更高境界？如果有，为何世人不知？"
- 问题按优先级排序：
  - **关键缺失**（影响故事合理性的必答项）
  - **建议补充**（能丰富世界层次的推荐项）
  - **可选拓展**（锦上添花的细节）
- 用户可以用自然语言回答，AI 自动将回答整合进世界设定文档
- 支持用户主动提问："如果我的世界有 X 规则，那 Y 会怎样？"AI 推演逻辑后果

---

### 阶段 6：一致性校验 & 冲突检测（Consistency Check）

**目的**：确保世界设定自洽，发现逻辑矛盾。

- **规则冲突检测**：
  - 核心公理 vs 维度细节（如公理说"无魔法"，但文化层提到了魔法学院）
  - 维度间矛盾（如科技水平是"中世纪"，但武器描述中出现了"激光剑"）
- **合理性评估**：
  - 经济系统是否能运转（货币、贸易、资源循环）
  - 权力结构是否稳定或有明确的不稳定理由
  - 人口/资源比是否合理
- **故事可行性评估**：
  - 当前世界设定是否能支撑足够的冲突和故事空间
  - 是否有"死胡同"设定（写到后期无法自圆其说）
- 输出一致性报告卡，标注 ✅ 通过 / ⚠️ 警告 / ❌ 矛盾，用户可逐条处理

---

### 阶段 7：世界观总览 & 可视化（Overview & Visualization）

**目的**：让用户一目了然地看到完整世界全貌。

- **世界观设定文档**：按维度分章节的结构化文档，可折叠/展开
- **势力关系图**：各势力/种族/组织之间的关系（同盟、对立、从属）
- **力量体系图**：修炼/科技等级的金字塔可视化
- **地理概念图**：简单的区域关系示意（不需要精美地图，用节点图表示即可）
- **历史时间线**：可交互的时间轴，标注关键事件
- **世界观概况卡片**：一张浓缩的"名片"，用几句话总结核心特色

---

### 阶段 8：精炼 & 迭代（Refinement & Iteration）

**目的**：在总览的基础上，用户可对任何部分进行深度打磨。

- 对任意维度发起"精炼"请求（复用已有的 refine API，前端补充 UI）
- 精炼等级：
  - **Light**（润色）— 保持核心不变，优化表述、补充细节
  - **Deep**（深挖）— 大幅扩展，增加子维度、案例、故事钩子
- 支持"替代方案"模式：AI 生成 2-3 个不同方向的设定，用户选择最喜欢的
- 修改后自动触发局部一致性检查（仅检查受影响的关联维度）

---

### 阶段 9：世界观关联小说 & 角色（World-Novel-Character Integration）

**目的**：世界观不是孤立存在的，需要与小说创作流程打通。

- **World ↔ Novel 关联**：
  - 数据库增加 `Novel.worldId` 外键
  - 创建小说时可选择已有世界观
  - 小说写作时自动将世界设定注入 LLM 上下文（作为 System Prompt 的一部分）
- **World ↔ Character 关联**：
  - 角色创建时参考世界规则（如修炼体系等级决定角色实力上限）
  - AI 检查角色背景是否与世界设定吻合
- **世界观约束写作**：
  - 章节生成时，AI 可访问世界观数据，避免出现与设定矛盾的内容
  - 配合已有的 `ConsistencyFact` 和 `NovelBible` 机制，将世界规则转化为约束条件

---

### 阶段 10：导出 & 版本管理（Export & Versioning）

**目的**：保护用户的创作成果，支持协作和备份。

- **导出格式**：
  - Markdown 文档（按维度分章节）
  - JSON 结构化数据（方便程序化使用）
  - PDF（精排版的世界观设定集）
- **版本管理**：
  - 每次重大修改自动创建快照（snapshot）
  - 支持版本对比（diff）和回滚
  - 版本标签功能（如"v1-初稿"、"v2-定稿"）
- **导入**：
  - 从 Markdown / JSON 导入已有世界设定
  - 从自然语言文本中 AI 提取世界观要素并结构化

## 三、素材库激活方案（WorldPropertyLibrary）

当前 `WorldPropertyLibrary` 表已有 `name`, `description`, `category`, `worldType`, `usageCount` 字段，建议按如下方式激活：

### 素材分类

| category | 示例 |
|----------|------|
| `terrain` | 浮空岛、地下城、虫洞、深渊裂谷 |
| `race` | 精灵族、龙人、机械种族、灵魂体 |
| `power_system` | 斗气、灵力、源力、纳米机器人 |
| `organization` | 暗杀组织、商会联盟、修炼学院 |
| `resource` | 灵石、魔晶、暗物质、世界树果实 |
| `event` | 天地大劫、种族战争、觉醒事件 |
| `artifact` | 上古神器、禁忌之书、空间之钥 |
| `custom` | 用户自定义分类 |

### 使用方式

- 用户在逐步构建维度时，可从素材库中拖拽/选择现成元素加入世界
- AI 根据世界类型推荐相关素材
- 用户可将自己世界中的优秀设定存入素材库，供未来复用
- `usageCount` 自增记录热门素材，用于排序和推荐

## 四、数据模型改进建议

### World 表扩展

```prisma
model World {
  id          String   @id @default(cuid())
  name        String
  description String?
  worldType   String?          // 世界类型（玄幻/科幻/...）
  axioms      String?          // 核心公理（JSON 数组）
  background  String?
  geography   String?
  cultures    String?
  magicSystem String?
  politics    String?
  races       String?
  religions   String?
  technology  String?
  conflicts   String?
  history     String?          // 新增：历史时间线
  economy     String?          // 新增：经济体系
  factions    String?          // 新增：势力关系（JSON）
  status      String  @default("draft")  // draft / refining / finalized
  version     Int     @default(1)
  novels      Novel[]          // 新增：关联小说
  snapshots   WorldSnapshot[]  // 新增：版本快照
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model WorldSnapshot {
  id        String   @id @default(cuid())
  worldId   String
  world     World    @relation(fields: [worldId], references: [id], onDelete: Cascade)
  label     String?
  data      String   // 完整世界观 JSON 快照
  createdAt DateTime @default(now())
}
```

### Novel 表增加关联

```prisma
model Novel {
  // ... 现有字段
  worldId  String?
  world    World?  @relation(fields: [worldId], references: [id], onDelete: SetNull)
}
```

## 五、LangGraph 工作流改进建议

当前 `worldBuildingGraph.ts` 为 stub，建议改造为真正的多节点 LangGraph 工作流：

```
[SeedNode] → [AxiomNode] → [GeographyNode] → [PowerSystemNode]
                                ↓
                        [SocietyNode] → [CultureNode] → [HistoryNode]
                                                            ↓
                                                    [ConflictNode] → [ConsistencyCheckNode]
                                                                            ↓
                                                                    [SummaryNode]
```

每个节点：
- 接收上游节点的输出作为上下文
- 调用 LLM 生成本层维度内容
- 输出结果 + 一致性标注
- 支持 human-in-the-loop：每个节点输出后暂停，等待用户确认或修改后再继续

## 六、优先级排序建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 分层逐步构建 + 维度可选 | 核心体验升级，替代一次性生成 |
| P0 | 核心规则定义 | 确保后续生成有约束，大幅提高质量 |
| P0 | World ↔ Novel 关联 | 打通创作流程闭环 |
| P1 | 一致性校验 | 提升设定质量，减少逻辑漏洞 |
| P1 | 精炼 UI（前端接入 refine API） | 后端已就绪，前端补上即可 |
| P1 | AI 交互式问答 | 显著提升世界深度 |
| P1 | 类型模板系统 | 降低上手门槛 |
| P2 | 素材库激活 | 提升复用效率 |
| P2 | 世界观总览 & 可视化 | 改善信息呈现 |
| P2 | 版本管理 | 保护创作成果 |
| P3 | 导入/导出 | 便捷但非核心 |
| P3 | LangGraph 多节点改造 | 架构优化，中长期收益 |
